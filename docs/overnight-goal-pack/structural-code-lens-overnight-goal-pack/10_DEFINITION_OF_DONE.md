# Definition of Done

A task is done only when implementation exists, tests exist, tests were run, evidence was recorded, browser behavior was checked when relevant, no secret or hardcoding was introduced, and logs were updated.

The run is done when:
- phases are marked complete, partial, blocked, or deferred;
- final commit and working-tree state are recorded;
- screenshots exist;
- test matrix is updated;
- AI provider status is recorded;
- morning report clearly lists remaining work.
