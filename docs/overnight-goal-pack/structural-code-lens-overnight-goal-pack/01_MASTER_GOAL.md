# Master Goal

## Final target

A learner can paste supported Python, provide inputs, press Visualize, and receive synchronized Flow, State, Guided Trace, and Graph Inspector views derived from verified execution. The learner can ask an environment-selected AI provider to explain the current step or whole process using the Language of Learning vocabulary.

## Required outcomes

- Flow is the default Decode view.
- Views are ordered Flow, State, Guided Trace, Graph Inspector.
- Fresh pasted programs work end to end.
- Unseen renamed variations prove structural derivation.
- Generic supported behavior differs from unsupported behavior.
- No fixture-specific or value-specific learner output.
- AI provider selection is environment-driven.
- DeepSeek works through an OpenAI-compatible adapter.
- Mock, OpenAI, Anthropic, and generic OpenAI-compatible seams exist.
- AI answers are grounded in server-verified semantic facts.
- Deterministic explanation fallback remains available.
- Real-user browser tests and screenshots exist.
- Decision, progress, blocker, and morning-review logs are completed.

## Success criteria

1. Six supported pasted programs and one unsupported program pass browser tests.
2. Three unseen variations work without code-specific hacks.
3. Flow, State, and Guided Trace agree at tested steps.
4. AI does not invent values, branches, returns, or support status.
5. Changing provider requires environment changes, not learner-UI edits.
6. Product still teaches when AI is disabled.
