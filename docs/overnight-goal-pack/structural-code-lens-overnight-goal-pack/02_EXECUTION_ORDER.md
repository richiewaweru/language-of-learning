# Execution Order

## Phase 0 — Baseline
Record branch, commit, environment, current screenshots, and all test results.

## Phase 1 — Audit
Inspect analyzer, trace, semantic normalization, projections, symbol registry, Decode, lesson routes, AI seams, and tests. Search for hardcoded names, results, indexes, step counts, fixture routing, and duplicated trace interpretation.

## Phase 2 — Decode polish
Make Flow default; reorder tabs; hide unfinished upload; improve loading, empty, invalid-input, API-failure, unsupported, and retry states; remove developer clutter; preserve useful state after errors; verify responsive behavior.

## Phase 3 — Pasted-code proof
Add browser tests for accumulation, count, linear search, early return, mutation, swap, binary search, generic supported, and unsupported. Add renamed unseen variations. Remove discovered source-specific hacks.

## Phase 4 — Flow polish
Connect selected collection cell to current item, show movement, animate old-to-new state, delay result until return, improve hierarchy, prevent overflow, support reduced motion and mobile.

## Phase 5 — State polish
Show role, name, current value, previous value, and change. Highlight changed rows, show cursor advancement, result pending, frame grouping, and alias identity.

## Phase 6 — Guided Trace polish
One primary event, active source range, values involved, learner-facing explanation, compact event glyph, short history, no full graph, no horizontal overflow.

## Phase 7 — Graph Inspector polish
Keep advanced, focus active neighborhood, fade inactive content, improve zoom, minimap, legend, and labels.

## Phase 8 — AI runtime
Create provider-neutral contracts, config, gateway, capability routing, mock provider, OpenAI-compatible provider, OpenAI seam, Anthropic seam, retries, timeouts, safe status.

## Phase 9 — AI teaching layer
Server re-verifies source and args, builds TeachingContext, implements explain-step, explain-program, scoped chat, structured response validation, and deterministic fallback.

## Phase 10 — Ask Lens UI
Add step explanation, whole-process explanation, chat, step attribution, context-change handling, loading, retry, fallback, responsive and accessible behavior.

## Phase 11 — AI evaluation
Run grounding, unsupported honesty, schema, vocabulary, fallback, latency, and provider-routing evaluations.

## Phase 12 — Class route
Fully polish accumulation, linear search, and array mutation with consistent lesson anatomy, prediction, variation, and transfer checks.

## Phase 13 — Handoff
Run all tests, capture screenshots, update logs and matrix, record final commit and remaining risks.
