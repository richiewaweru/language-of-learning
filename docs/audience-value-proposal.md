# Audience Value Demonstration — Product and Build Proposal

**Working phase:** V — Audience Value Proof  
**Repository:** richiewaweru/language-of-learning

## Primary objective

Transform the existing technical proof into a polished experience that allows a learner, teacher, collaborator, or investor to understand the product's value without requiring a verbal explanation from the creator.

## Central promise

> See what code is doing, not just what it says.

**Supporting statement:** Language of Learning turns code into a synchronized visual model of values, decisions, state changes, and results.

## First-time visitor goals

1. Understand the problem within 30 seconds.
2. Experience the product within one minute.
3. Understand the visual model within three minutes.
4. Recognize the educational value within five minutes.
5. See that the system can grow beyond one example.

## Non-negotiable principles

1. **Not a generic code animator** — every visible value, state change, branch, and return comes from deterministic engine artifacts.
2. **Teaches structures, not isolated examples** — visitors recognize reusable patterns.
3. **Motion explains meaning** — no decorative animation.
4. **Learner language first** — engineering vocabulary only in optional technical sections.
5. **No AI for execution truth** — deterministic templates only.
6. **No broad DSA expansion** during V0–V4.

## Phases

| Phase | Goal |
|-------|------|
| V0 | Product framing and information architecture |
| V1 | Public landing page and flagship demonstration |
| V2 | Flagship four-lesson pathway |
| V3 | Connected Truth View |
| V4 | Validation, polish, and public-demo readiness |

## Definition of success

A first-time visitor can say: *"This shows me how the values move and how the program builds its result. I can also see that different-looking loops share the same underlying structure."*

Success requires both **ENGINE TRUTH + AUDIENCE COMPREHENSION**.
