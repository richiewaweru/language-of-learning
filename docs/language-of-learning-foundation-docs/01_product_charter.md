# Product Charter

## Working name

**Language of Learning**

Core engine and first product surface:

**Structural Code Lens**

## Mission

Make difficult technical ideas easier to understand by exposing their recurring structures, transformations, state changes, and variations through a consistent interactive visual language.

## Product thesis

Many learners can understand a concept once its structure is visible, but struggle when the same concept is presented only as dense text, notation, or code. The product should reduce the hidden work required to reconstruct that structure.

The system does not claim that people have fixed learning styles. It selects and coordinates representations according to the material, learner knowledge, linguistic context, and current misunderstanding.

## User problems

### Decode
“I can read the lines, but I cannot see how the parts connect.”

### Learn
“I understand examples individually, but I cannot recognize the recurring pattern.”

### Studio
“I can create an explanation, but I cannot reliably produce consistent, correct, reusable interactive learning material.”

## First flagship topic

**How loops build results**

- collection;
- current item;
- mutable state;
- accumulation;
- count;
- filter;
- transform.

## Non-goals for v0.1

- multiple programming languages;
- repository-scale understanding;
- automatic publication;
- arbitrary AI-generated diagrams;
- complete DSA coverage;
- fixed learning-style labels;
- unrestricted Python execution;
- a general textbook generator.

## Success definition

A novice can paste a supported Python function and identify its structure, step through execution, explain state changes, name the larger pattern, and solve a transfer problem.

An author can create, verify, and publish one complete learning lesson using the same artifacts.
