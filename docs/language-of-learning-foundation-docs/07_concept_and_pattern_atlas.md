# Concept and Pattern Atlas

## Purpose

The atlas guides what the system teaches. It is broader than the parser ontology.

## Four levels

### Semantic primitives
Value, binding, collection, operation, sequence, branch, repetition, function, call, return, mutation, effect.

### Language constructs
Variable, parameter, argument, expression, conditional, loop, list, object, class, method, module, exception.

### Recurring patterns
Accumulate, count, filter, transform, search, validate, guard, pipeline, callback, request-response.

### Algorithms and system structures
Traversal, two pointers, sliding window, divide and conquer, recursion tree, dynamic-programming state transition, graph search, heap selection, event-handler-state-render.

## Admission test

A concept is added only when it:

1. represents a distinct useful mental operation;
2. cannot be reduced without losing explanatory value;
3. recurs across examples;
4. supports a stable visual law;
5. improves explanation, prediction, or transfer;
6. has fixtures and checks.

Cross-language occurrence is evidence, not an absolute rule.

## v0.1 pattern definitions

- ACCUMULATE: build up one result.
- COUNT: increase a running number.
- FILTER: keep selected original values.
- TRANSFORM: change every value.
- SEARCH: stop at the first match.
- GUARD: reject invalid input early.

## Versioning

Every concept revision records atlas version, visual grammar version, canonical examples, misconceptions, prerequisites, and verification date.
