# Authoring, Verification, and Publishing

## Studio workflow

```text
Select concept
→ attach canonical examples
→ run analyzer and trace
→ generate scenes
→ draft explanations and checks
→ machine verification
→ human review
→ publish revision
```

## Recommended lesson structure

1. Hook or problem.
2. Canonical mental model.
3. Structural scene.
4. Code mapping.
5. Execution trace.
6. Variation family.
7. Misconception contrast.
8. Prediction check.
9. Reconstruction task.
10. Transfer problem.
11. Next-concept bridge.

## Machine checks

Before human review:

- schemas validate;
- source parses;
- sample call executes;
- result matches;
- unsupported regions are expected;
- pattern rule fires;
- scene references real nodes;
- source mappings resolve;
- layout has no overlaps;
- backward stepping restores state;
- captions use verified parameters;
- keyboard controls work;
- reduced-motion fallback works;
- static fallback exists.

## Human review

The reviewer confirms conceptual accuracy, useful visual structure, non-misleading analogies, real variation families, realistic misconceptions, accessible wording, syntax-to-shape mapping, valid assessment, and genuine transfer.

## Publishing states

```text
draft → review → machine_checked → human_verified → published → superseded → archived
```

Published revisions are immutable. Corrections create a new revision.
