# Delivery Roadmap

## Phase 0 — Decisions and contracts

Product charter, scope, semantic contract, visual constitution, ADRs, repo skeleton, and CI.

Exit: schemas compile and governing decisions are approved.

## Phase 1 — Analyzer

AST parser, stable IDs, semantic graph, unsupported regions, graph inspector, and fixtures.

Exit: ten unseen beginner snippets match manual expectations.

## Phase 2 — Trace runtime

Sample-call editor, bounded execution, binding snapshots, mutation, branch, loop, return, and effect events.

Exit: canonical and hostile fixtures pass.

## Phase 3 — Pattern engine

Deterministic ACCUMULATE, COUNT, FILTER, TRANSFORM, SEARCH, and GUARD.

Exit: positive and negative tests pass.

## Phase 4 — Scene and visual engine

Layout, visual primitives, selection synchronization, controls, accessibility, accumulator and filter slices.

Exit: pasted variations work without manual coordinates.

## Phase 5 — Decode alpha

Public paste interface, code/shape/trace/pattern views, unsupported panel, grounded explanation, transfer check, optional save, minimal telemetry.

Exit: a novice completes the journey without assistance.

## Phase 6 — Studio

Concept registry, examples, lesson composer, preview, machine checks, human approval, revisions, publishing.

Exit: one lesson moves from draft to published.

## Phase 7 — Learn

First verified track, interactive pages, static fallback, navigation, progress, transfer assessment, SEO.

Exit: five to eight verified lessons are live.

## Phase 8 — Expansion

Recursion, linked lists, stacks, queues, trees, graph search, sorting, complexity, repository import.

Expansion follows evidence.
